Preparation

Hardware Needed for Set-up

Required Equipment

  • A pair of Bat-867 Wireless AP devices with the exact same firmware installed on them.
    Note If the devices are running different firmware versions, several issues will occur while trying to configure them within the LanConfig software.
  • A diagnostic laptop and a standard Cat5e/Cat6 network cable to directly connect to the devices.
  • The Machine Name where the pair of APs will be installed. This will be used as the P2P Radio ID. This must be 100% unique for each pair of Access Points being configured (using the TARUS machine name is standard practice).
  • Your initial Wi-Fi Channel selection. The 5GHz band is strongly recommended. Channel choice is critical. If this is your first pair of devices, choose a channel and work your way up 149, 153, 157, 161...
    Note This channel will likely need to be changed once the machine is installed onsite to avoid overlapping with the customer's factory network.
  • Verify paddle antennas are connected before powering on the devices!
System

Resetting a Device

Factory Reset Procedure

If you are working with an AP that was previously configured and need to start fresh, follow these steps to wipe it:

  1. Locate the small reset pinhole just below the link lights on the face of the AP.
  2. Using a paperclip or small tool, press and hold the reset button for 20 seconds or until the status lights turn solid red.
  3. Keep holding the button until the light turns red for a second time.
  4. Release the button. The unit is now wiped, has no IP address, and no name. Proceed to Initial Setup.
Network

Initial Setup

1

Network Concept & Rules

  • Both APs in the pair must be on the same subnet with the same netmask. (Example: 10.0.0.xxx / 255.255.255.0).
  • The system is configured purely as a point-to-point wireless bridge.
  • You must designate one unit as the Master (Desk) and one unit as the Slave (Tower).
  • To bridge successfully, the devices reference each other using unique P2P Radio IDs. Master: 4001A Desk | Slave: 4001A Tower
2

PC Preparation

  1. Open Windows Network Adapters, right-click your Ethernet connection, select Properties, and choose IPv4.
  2. Set a static IP address for your laptop's physical NIC: IPv4 IP address: 10.0.0.50 and Subnet mask: 255.255.255.0
  3. Connect your laptop directly to the first AP with a network cable.
  4. Open the Hi-Discovery program from your Windows Start Menu.
3

Assigning the IP in Hi-Discovery

In the Hi-Discovery software, click the Scan button. Once your connected AP appears in the list, double-click it. Enter the exact information below based on which unit you are holding:

  • Master Unit: Name = master | IP Address = 10.0.0.1 | Netmask = 255.255.255.0
  • Slave Unit: Name = slave | IP Address = 10.0.0.2 | Netmask = 255.255.255.0
Note Click OK in the dialog box. The software will write this configuration to the AP. You can now close the Hi-Discovery application.
Configuration

LanConfig Program

1

Find the Device in LanConfig

  1. Open the Hirshmann LanConfig program from your Start Menu.
  2. If the device is already listed in the main window from a previous setup, verify the IP address matches what you just set in Hi-Discovery (10.0.0.1 or 10.0.0.2).
  3. If the main window is empty, click FILE in the top-left corner and select Find Devices. Once the scan finds your device, click Add Selected devices to bring it into the main view.
2

Change the Default Administrator Password

  1. Right-click the device in the list and choose Configure.
  2. A prompt will appear requesting credentials. Use the factory defaults: Administrator: admin, Password: private. Click Okay.
  3. In the new configuration window, look at the folder tree on the left side. Expand the Management folder.
  4. Click on Admin.
  5. In the main panel, enter a new Main Device Password (e.g. Passme123) and repeat it in the confirmation box.
  6. Click OK at the bottom of the window to save the password change.
3

Run the Basic Settings Wizard

Double-click the device in the main window to open the Setup Wizard menu, then select Basic Settings. Click Next after completing each of the following screens:

  1. Device name screen: Leave this exactly as it shows (it pulled "master" or "slave" from Hi-Discovery).
  2. Allow device access: Ensure this is set to Local networks.
  3. DHCP screen: Ensure DHCP mode is set to Off.
  4. Set IP screen: Verify the IP is correct. Do not enter a gateway or DNS server.
  5. Time Zone: Set to -5 (EST) and set DST to Automatic – USA. Ensure Time Server is pool.ntp.org.
    Note Adjust this based on the physical timezone where the machine will reside.
  6. Optional info: Leave these fields blank.
  7. Updating: Select No automatic updating.
Note Click Finish. The configuration will upload, and the wizard select screen will reappear.
4

Configure WLAN & Point-to-Point

When the wizard menu re-opens, click Configure WLAN, then select Step by Step detailed configuration of WLAN.

  1. Screen 1: Country Selection: Set the Country to USA. Click Next.
  2. Screen 2: WLAN Operating Mode: Set the WLAN operating mode to Access Point. Click Next.
  3. Screen 3: Frequency Band & Channel: Select the 5GHz (802.11a/n) band. Choose your specific channel from the dropdown. Recommended Channels: Use 40, 44, 48 or 165.
    Note Rule: The Master and Slave MUST be on the exact same channel to communicate.
  4. Screen 4: Antenna Type: Select the top radio button for The provided default respectively internal antennas are used. Click Next.
    Note If a subsequent screen asks for Antenna Gain, leave it at the default of 3.
  5. Screen 5: Setup Type: Select the bottom radio button for Configure WLAN Point-to-Point remote site. Click Next.
  6. Screen 6: P2P Network Settings: Point-to-Point operation mode: Select Exclusive. P2P Radio ID: Enter the unique name (e.g., 4001A Desk). Channel Selection Scheme: Choose Master or Slave. Do not forward...: Ensure this box is Left Unchecked.
  7. Screen 7: Maximum Distance: Enter 0 km. Click Next.
  8. Screen 8: Encryption Security: Select Use WPA-2 (802.11i) Enhanced Security. Click Next.
  9. Screen 9: WPA-2 Passphrase: Enter the exact same password for both devices. Development default: Passme123. Click Next.
  10. Screen 10: Point-to-Point Remote APs: Ensure the dropdown says exactly P2P-1-1 is off, MAC: empty. Click Next.
  11. Screen 11: Target the Remote Device: Check the box for Enable this Point-2-Point channel. Select the radio button for Recognize remote site by: P2P radio ID. P2P radio ID: Type in the exact name of the OTHER device (e.g. 4001A Tower). Click Next.
Note Click Finish. Once this process is fully completed on both devices, their physical WLAN status lights should illuminate flashing Green. Repeat Steps 1-4 for the second device in the pair.
5

Performance & Stability Fine-Tuning

These advanced settings are not available in the wizard and must be changed manually on BOTH devices after initial setup is complete.

Part A: Physical WLAN Settings

  1. In LanConfig, right-click the device and select Configure.
  2. In the left sidebar tree, expand the Wireless Lan folder, then click on General.
  3. In the main panel, click the button labeled Physical WLAN settings.
  4. A new window opens. Click the Radio tab at the top.
  5. Verify your channel sub-band mapping. Depending on the channel chosen, select either Sub-band 1 or Sub-band 2 from the dropdown.
    Note For North America, use only these 5GHz Channels: Sub-band 1 (36, 40, 44, 48) or Sub-band 2 (149, 153, 157, 161, 165).
  6. Change Max. channel bandwidth to exactly 20 MHz and Antenna grouping to exactly Antenna 1.
  7. Click OK to close the Physical settings window.

Part B: Logical WLAN Settings

  1. Still on the Wireless Lan > General page, click Logical WLAN settings.
  2. Select the very first option: WLAN interface 1 - Network 1 (On).
  3. On the Network tab, change Client Bridge Support to NO.
  4. Click the Transmission tab at the top.
  5. Change the RTS Threshold value to exactly 65.
  6. Click OK on all open windows to write these final configurations.
6

Final Validation & Ping Test

You have completed the baseline setup. To verify the wireless bridge is passing data, perform this physical ping test:

  1. Power on both the Master and Slave Hirschmann units.
  2. Change your diagnostic laptop's IPv4 address to a completely different subnet than what you just used (e.g. change it to 192.168.1.50).
  3. Plug your laptop's ethernet cable into the Master (Desk) AP.
  4. Open the Windows Command Prompt (cmd).
  5. Attempt to ping the Slave AP by typing: ping 10.0.0.2 and hitting Enter.
  6. If you receive replies, your laptop is successfully sending data through the Master, across the air to the Slave. The bridge is functional.
Hardware

LED Status Indicators

Status Lights Diagnostics

There are 3 main flashing status lights on the faceplate of the Hirschmann unit. Use these for quick visual diagnostics:

"P" (Power & Status)

  • Solid Green: Booted and normal operation.
  • Flashing Red: Hardware error detected. Check diagnostics.
  • Flashing Green/Red: Security warning: The default password 'private' is still active.

"LS/DA" (Link Data)

  • Off: No physical ethernet cable detected.
  • Solid Green: Ethernet connected but idle.
  • Flashing Yellow: Active network data traffic is flowing.

"WLAN" (Radio Link)

  • Off: Radio is disabled or configuration is incomplete/failed.
  • Solid Green: WLAN module activated and actively broadcasting the bridge.
Diagnostics

Using the Ekahau Sidekick 2

Mapping Customer Network Traffic

The Ekahau Sidekick 2 is a professional RF spectrum analysis tool. Follow these steps to locate an unused 5GHz channel for the Hirschmann APs:

  1. Connect & Launch: Power on the Ekahau Sidekick 2 device. Connect it directly to your diagnostic iPad or Laptop using the provided USB-C cable. Launch the Ekahau Analyzer app (Mobile) or Ekahau Pro software (Desktop). Sign in with your Tarus Ekahau credentials if prompted.
  2. Read the Spectrum: In the app, navigate to the Spectrum Analysis or Waterfall view. Filter the view to show only the 5GHz band. Observe the color densities: Red/Orange peaks mean a channel is heavily utilized. Deep Blue/Black means the channel is empty and available.
  3. Walk the Machine Footprint: Hold the Sidekick device and physically walk around the entire perimeter of where the TARUS machine is installed. Watch the analyzer for a channel (e.g. 149 or 153) that remains completely "quiet" (Blue/Black).
  4. Select & Document: Identify the cleanest 20MHz wide channel in the 5GHz spectrum. Write down this channel number to program into the Hirschmann LanConfig software.
Deployment

Site Deployment & Final Tuning

Mitigating Channel Interference

When deploying a machine onsite, the pre-programmed development channel will almost always overlap with the customer's enterprise Wi-Fi. You must execute the following protocol at every installation.

Quick RF Cheat Sheet

  • Channels 40, 44, 48 (U-NII-1): Immune to DFS sweeps, ensuring zero sudden communication dropouts. Middle 5GHz channels share space with radar and can shut down instantly.
  • Channel 165 (U-NII-3 Emergency Hatch): The 20MHz "Un-bondable" Island. Corporate IT autotuning systems almost completely ignore it, keeping it a safe fallback zone.

Deployment Steps

  1. Identify a Clean Channel: Use the Ekahau Sidekick to map the RF space and document an empty 5GHz channel (prioritizing 40, 44, 48, or 165).
  2. Update Both Hirschmann Devices: Right-click device -> Configure -> Wireless Lan -> General -> Physical WLAN settings. On the Radio tab, change the Channel Number dropdown to the clean channel. Click OK to save. Repeat for the 2nd device.
  3. Mandatory Customer I.T. Coordination: You MUST contact the customer's internal I.T. department. Request that they explicitly "Exclude" or "Lock Out" your chosen 5GHz channel from their ceiling Access Points in the immediate area.
    Note Enterprise networking systems use "Auto-RF" tuning. If I.T. does not lock out your channel, their system will eventually auto-tune itself onto your clean channel, overpowering the Hirschmann radios.
